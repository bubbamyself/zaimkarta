## 1. Контекст

ZaimKarta — SEO-витрина займов и финансовых офферов с CPA-монетизацией. Пользователь приходит на сайт, смотрит подборки, карточки, SEO-страницы и инструменты, затем переходит к кредитору через внутренний маршрут `/go/[slug]`, где фиксируются `lead_id`, `click_id` и `OfferClick`.

В CPA-вертикали микрозаймов у рекламодателей часто есть стоп-регионы: заявки от пользователей с регистрацией в определенных субъектах РФ не рассматриваются или почти не одобряются. Сейчас ZaimKarta может показать пользователю оффер, который фактически не подходит ему по региональным условиям кредитора.

## 2. Цель фичи

Добавить учет региона регистрации пользователя и региональных ограничений офферов.

Задача:

- пользователь выбирает регион регистрации;
- админ указывает стоп-регионы в оффере;
- ZaimKarta скрывает офферы, которые ограничены для выбранного региона;
- `/go/[slug]` дополнительно защищает CPA-переход на сервере.

Ожидаемый эффект:

- повысить качество CPA-лидов;
- снизить количество бесполезных переходов;
- улучшить пользовательский опыт;
- повысить доверие к витрине;
- экономить время пользователя;
- не отправлять рекламодателю лиды, которые заведомо не проходят его региональные условия.

## 3. Терминология

Использовать:

- `регион регистрации`;
- `региональные ограничения`;
- `условия кредитора`;
- `предложения, которые с большей вероятностью подходят`.

Избегать:

- `бан`;
- `черный список`;
- `запрещенный регион`;
- `отказной регион`;
- `вам не одобрят`;
- `этот регион заблокирован`.

## 4. Справочник регионов

Админ предоставит список регионов в формате:

```text
55 Омская область
54 Новосибирская область
38 Иркутская область
...
```

Этот список нужно превратить в справочник регионов РФ.

Рекомендуемый файл:

```text
site/src/lib/russian-regions.ts
```

Рекомендуемый формат:

```ts
export type RussianRegion = {
  code: string;
  number: string;
  name: string;
};

export const RUSSIAN_REGIONS: RussianRegion[] = [
  {
    code: "55",
    number: "55",
    name: "Омская область",
  },
  {
    code: "54",
    number: "54",
    name: "Новосибирская область",
  },
  {
    code: "38",
    number: "38",
    name: "Иркутская область",
  },
];
```

На первом этапе цифровой код региона можно использовать как основной `code`.

Важно:

- хранить код региона, а не только название;
- название использовать для отображения;
- код использовать для фильтрации и сохранения в базе;
- если название региона позже уточнится, сохраненные ограничения не должны сломаться.

Нужны helper-функции:

```ts
getRussianRegionByCode(code)
isRussianRegionCode(code)
normalizeRegionCodes(codes)
```

## 5. Данные

В модель `Offer` добавить поле:

```prisma
restrictedRegionCodes String[] @default([])
```

Смысл:

- пустой массив — оффер показывается для всех регионов;
- массив кодов — оффер не показывается пользователям, выбравшим один из этих регионов;
- пример: `["55", "54", "38"]`.

Перед сохранением:

- принимать только коды из справочника;
- удалять дубли;
- неизвестные коды не сохранять.

## 6. Админка оффера

В форме создания и редактирования оффера добавить блок:

```text
Региональные ограничения
```

Описание блока:

```text
Выберите регионы регистрации, для которых этот оффер не должен показываться на витрине.
```

Дополнительная подсказка:

```text
Если регион пользователя есть в этом списке, оффер будет скрыт на сайте и не будет доступен для перехода через CPA-ссылку.
```

## 7. UI выбора регионов в админке

Нужен выпадающий список/комбобокс с поиском и чекбоксами.

Требования:

1. Placeholder:

```text
Найти регион
```

2. Поиск работает по:

- цифровому коду региона;
- названию региона.

Примеры:

- `55` находит `Омская область`;
- `омск` находит `Омская область`;
- `новос` находит `Новосибирская область`;
- `38` находит `Иркутская область`.

3. Строки списка:

```text
[ ] 55 — Омская область
[ ] 54 — Новосибирская область
[ ] 38 — Иркутская область
```

4. При клике на чекбокс регион добавляется в стоплист оффера.

5. При повторном клике регион удаляется из стоплиста.

6. Выбранные регионы показываются под полем как список или чипы:

```text
55 — Омская область ×
54 — Новосибирская область ×
```

7. У каждого выбранного региона есть удаление `×`.

8. Есть действие:

```text
Очистить выбранные регионы
```

9. Если ничего не выбрано, показывать текст:

```text
Региональных ограничений нет — оффер показывается для всех регионов.
```

## 8. Создание оффера

При создании нового оффера:

- `restrictedRegionCodes` по умолчанию пустой;
- админ может сразу добавить стоп-регионы;
- после сохранения выбранные коды сохраняются в базе.

## 9. Редактирование оффера

При открытии существующего оффера:

- админка читает `restrictedRegionCodes`;
- сохраненные регионы автоматически отмечены чекбоксами;
- выбранные регионы отображаются под полем;
- админ может добавить новые регионы;
- админ может удалить старые регионы;
- после сохранения список полностью обновляется.

## 10. Отображение в списке офферов админки

Для MVP необязательно, но желательно добавить краткий индикатор:

```text
Региональные ограничения: 3
```

или:

```text
Стоп-регионы: 55, 54, 38
```

## 11. Пользовательский попап

При первом заходе пользователь видит попап выбора региона регистрации.

Рекомендуемый текст:

```text
Здравствуйте! Выберите регион вашей регистрации — так мы покажем предложения, которые с большей вероятностью подходят вам по условиям кредиторов.
```

Требования:

- список регионов использует тот же справочник;
- пользователь выбирает один регион;
- выбор сохраняется;
- после выбора сайт показывает витрину с учетом фильтра;
- пользователь может изменить регион позже.

## 12. Хранение выбора пользователя

Рекомендуемый формат:

```text
zk_region_code=55
```

Рекомендация:

- использовать cookie `zk_region_code`;
- срок жизни 180 или 365 дней;
- `sameSite: "lax"`;
- cookie нужен серверу для `/go/[slug]`;
- фронту тоже нужно знать выбранный регион для UI.

## 13. Публичная фильтрация

Если пользователь выбрал регион:

```ts
offer.restrictedRegionCodes.includes(regionCode)
```

значит оффер скрывается.

Если регион не выбран:

- MVP-вариант: показать попап;
- до выбора можно показывать все офферы или мягко блокировать подбор;
- рекомендуемый вариант: не жестко блокировать сайт, но явно попросить выбрать регион для корректной витрины.

Фильтрация должна применяться:

- на главной;
- на SEO-страницах `/[slug]`;
- в списках офферов;
- в `home-offer-picker`;
- в `filterable-offers`;
- в SEO-инструментах, которые показывают офферы;
- в будущих витринах;
- в похожих/рекомендованных офферах на странице оффера, если такие есть.

## 14. Серверная защита `/go/[slug]`

Файл:

```text
site/src/lib/cpa-click.ts
```

Логика:

- прочитать `zk_region_code`;
- найти оффер;
- проверить `restrictedRegionCodes`;
- если код региона входит в ограничения оффера, не отправлять пользователя в CPA-сеть;
- сделать fallback redirect.

Возможный fallback:

```text
/?offer_unavailable_region=1
```

Позже можно показывать уведомление:

```text
Это предложение не подходит для выбранного региона. Мы покажем другие варианты.
```

## 15. Аналитика

В MVP можно не усложнять.

Позже обсудить:

- сохранять регион в `Lead`;
- сохранять регион в `OfferClick`;
- считать скрытия офферов по региону;
- передавать регион в CPA sub-параметр;
- строить отчет, какие регионы чаще всего фильтруют офферы.

## 16. Комплаенс

Важно:

- не писать, что ZaimKarta гарантирует одобрение;
- не писать, что человеку откажут;
- не писать, что регион запрещен;
- объяснять, что кредиторы могут иметь региональные условия;
- говорить о повышении релевантности предложений.

Безопасный текст:

```text
Выберите регион вашей регистрации — так мы покажем предложения, которые с большей вероятностью подходят вам по условиям кредиторов.
```

Текст для изменения:

```text
Регион регистрации: Омская область
Изменить
```

Текст для недоступного оффера:

```text
Это предложение не подходит для выбранного региона. Мы покажем другие варианты.
```
