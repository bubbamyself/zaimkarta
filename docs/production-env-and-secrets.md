# Production env и секреты ZaimKarta

Эта инструкция нужна перед первым запуском сайта на VPS. Файл `site/production.env.example` — только образец со специальными заглушками. Его нельзя использовать как настоящий production env без замены секретов.

## Что такое секреты

Секреты — это длинные уникальные «пароли-ключи». Человек обычно не вводит их при работе с сайтом: сервер читает их сам при запуске. Для MVP их не нужно регулярно менять. Гораздо важнее сделать их сильными, не положить в GitHub, не отправлять в чат и не потерять.

## Какие значения нужны

| Переменная | Секрет | Для чего нужна |
| --- | --- | --- |
| `NODE_ENV=production` | нет | Включает production-режим сайта |
| `NEXT_PUBLIC_SITE_URL=https://zaimkarta.ru` | нет | Сообщает сайту его публичный адрес |
| `POSTGRES_USER` | нет | Имя пользователя PostgreSQL |
| `POSTGRES_DB` | нет | Имя базы данных |
| `POSTGRES_PASSWORD` | да | Пароль PostgreSQL |
| `DATABASE_URL` | да | Полный адрес подключения к базе; внутри него есть пароль |
| `ADMIN_SESSION_SECRET` | да | Подписывает сессию админки и защищает ее от подделки |
| `LEAD_IP_HASH_SALT` | да | Защищает хеши IP-адресов посетителей |
| `ZAIMKARTA_MAIL_USER` | нет | Полный адрес рабочего ящика Beget |
| `ZAIMKARTA_MAIL_PASSWORD` | да | Даёт серверу доступ к письмам и отправке ответов |
| `ZAIMKARTA_MAIL_FROM_NAME` | нет | Имя отправителя в исходящих письмах |
| `ZAIMKARTA_MAIL_IMAP_HOST` / `ZAIMKARTA_MAIL_IMAP_PORT` | нет | Защищённое получение писем из Beget |
| `ZAIMKARTA_MAIL_SMTP_HOST` / `ZAIMKARTA_MAIL_SMTP_PORT` | нет | Защищённая отправка писем через Beget |

Регулярно руками человек вводит только пароль своей учетной записи в админке. `ADMIN_SESSION_SECRET`, `LEAD_IP_HASH_SALT` и `POSTGRES_PASSWORD` использует сайт или сервер автоматически.

`DATABASE_URL` должен содержать тот же настоящий пароль, что и `POSTGRES_PASSWORD`. Если пароль содержит специальные символы, их может потребоваться закодировать для URL. На этапе настройки VPS это нужно проверить отдельным подключением к базе.

Пароль `ZAIMKARTA_MAIL_PASSWORD` берётся из отдельной записи рабочего ящика в менеджере паролей. Он хранится только в закрытом env-файле VPS. Его нельзя добавлять в `site/.env.example`, `site/production.env.example`, Dockerfile, исходный код или переменные с префиксом `NEXT_PUBLIC_`.

## Как подготовить значения на Mac

Откройте Terminal и выполните:

```bash
openssl rand -base64 48
```

Выполните команду три раза и сохраните три разных результата:

1. для `ADMIN_SESSION_SECRET`;
2. для `LEAD_IP_HASH_SALT`;
3. для `POSTGRES_PASSWORD`.

Не используйте один результат сразу в нескольких местах. Не добавляйте к нему слова `test`, `local`, `example` или `change-this`: production-защита считает такие значения заглушками.

## Где хранить

- Сохраните значения в менеджере паролей как отдельную запись «ZaimKarta production».
- На сервере храните настоящий env-файл вне Git или в закрытой настройке окружения процесса.
- Ограничьте доступ к файлу владельцем серверного процесса.
- В Git храните только `site/.env.example` и `site/production.env.example` без настоящих значений.

## Чего нельзя делать

- Не коммитьте настоящий `.env` или `production.env` в GitHub.
- Не вставляйте секреты в задачи, чаты, письма, скриншоты и публичную документацию.
- Не используйте заглушки из example-файла и локальные пароли в production.
- Не помещайте секрет в переменную с префиксом `NEXT_PUBLIC_`: такие значения могут попасть в браузер.
- Не выводите `ZAIMKARTA_MAIL_PASSWORD` в логи и не передавайте его компонентам интерфейса.
- Не храните единственную копию только на VPS.

## Проверка перед запуском

1. Скопируйте `site/production.env.example` в настоящий закрытый env-файл на сервере.
2. Замените все `replace-with-...` на сгенерированные значения.
3. Убедитесь, что `DATABASE_URL` ведет в production PostgreSQL, а не на `localhost`.
4. Убедитесь, что `NEXT_PUBLIC_SITE_URL` равен `https://zaimkarta.ru`.
5. Проверьте через `git status`, что настоящий env-файл не предлагается добавить в Git.
6. Выполните `npm run lint` и production-сборку с настроенными переменными.
7. Откройте `/admin?section=email` под пользователем с ролью `BOSS` и проверьте входящие, тестовую отправку, спам и корзину.

Сайт в production должен отклонить отсутствующие, слишком короткие и очевидно тестовые `ADMIN_SESSION_SECRET` и `LEAD_IP_HASH_SALT`. Минимальная длина каждого — 32 символа. Локальная разработка при этом может использовать локальные значения.

## Если секрет утек

1. Считайте его скомпрометированным, даже если сообщение или commit уже удалены.
2. Сгенерируйте новое значение той же командой.
3. Замените его в менеджере паролей и на сервере, затем перезапустите сайт.
4. При утечке `ADMIN_SESSION_SECRET` все старые админские сессии после замены перестанут быть действительными.
5. При утечке `POSTGRES_PASSWORD` смените пароль в PostgreSQL и одновременно обновите `DATABASE_URL`.
6. Если секрет попал в Git, отдельно удалите его из истории или закройте доступ к репозиторию; одной правки последнего файла недостаточно.
7. При утечке `ZAIMKARTA_MAIL_PASSWORD` смените пароль ящика в Beget, обновите менеджер паролей и env на VPS, затем перезапустите приложение.
