# ZaimKarta SEO Research 2026

Дата проверки источников: 2026-07-20.

## 1. Что уже есть в проекте

ZaimKarta уже не пустой сайт и не простая витрина карточек. Это важно: следующий SEO-этап должен развивать существующую инфраструктуру, а не начинать ее заново.

Уже есть:

- публичные SEO-страницы `SeoPage` с типами `CATEGORY`, `ARTICLE`, `SERVICE`;
- статусы `DRAFT`, `PUBLISHED`, `PAUSED`, `ARCHIVED`;
- `contentBlocks` как структурированное тело страницы;
- связка страниц с офферами через `SeoPageOffer`;
- FAQ через `SeoPageFaqItem`;
- интерактивные инструменты через `SeoTool` и `SeoPageTool`;
- калькулятор переплаты и чек-лист заявки как первые SEO-инструменты;
- публичный CPA-переход через `/go/[slug]`;
- учет кликов через `OfferClick`;
- хлебные крошки и `BreadcrumbList` JSON-LD на публичных SEO-страницах;
- canonical для главной, SEO-страниц и страниц офферов;
- `/robots.txt` и `/sitemap.xml`;
- публичные страницы `/blog`, `/services`, правила и редакционная политика;
- production checklist с проверками `NEXT_PUBLIC_SITE_URL`, canonical, breadcrumbs, sitemap и robots.

Главный вывод: ближайшая точка роста не в новой архитектуре, а в правильной контентной системе, редакционном процессе, безопасных формулировках и измеримых SEO-экспериментах.

## 2. Что требуют поисковые системы

### Яндекс

Официальные материалы Яндекса сходятся в одном: сайт должен решать задачу пользователя, быть понятным, полезным, оригинальным и вызывать доверие.

Выводы для ZaimKarta:

- страницы про займы должны быстро отвечать на вопрос пользователя, а не быть набором SEO-фраз;
- финансовая тематика требует дополнительных сигналов надежности;
- условия, риски, ограничения, документы, ставка, ПСК и дата проверки должны быть видимыми;
- дубли и массовые страницы без пользы опасны;
- навигация, структура, readable URL, title, description, заголовки и Schema.org нужны как базовая гигиена;
- пользовательские сигналы и Метрика важны, но их нельзя имитировать, их нужно улучшать реальной пользой.

### Google

Google отдельно подчеркивает people-first content, борьбу с масштабной генерацией бесполезных страниц, scraping и site reputation abuse. Для ZaimKarta это означает: нельзя делать поток одинаковых AI-статей или страниц под каждый микроключ без собственной пользы.

Выводы для ZaimKarta:

- статья должна быть написана для человека, а не только для поисковика;
- нельзя копировать структуру и факты конкурентов без собственной проверки;
- автоматическая генерация большого количества страниц опасна;
- canonical, sitemap, breadcrumbs и structured data должны помогать поисковику понимать настоящую структуру сайта;
- внутренние ссылки должны вести на реальные полезные страницы, а не быть искусственной сеткой.

## 3. Группы спроса

| Группа | Пример интента | Лучший формат |
| --- | --- | --- |
| Коммерческий | `займы онлайн на карту`, `микрозайм срочно` | Подборка `CATEGORY` с офферами, фильтрами, рисками, FAQ |
| Сравнительный | `какой займ лучше`, `займер или екапуста` | Сравнение, таблица, статья или отдельная подборка |
| Информационный | `что такое пск`, `как работает микрозайм` | Статья `ARTICLE`, глоссарий, справочник |
| Проблемный | `просрочка в мфо что делать`, `мфо звонят родственникам` | Статья с пошаговым разбором и первоисточниками |
| Навигационный | `zaymer займ`, `е капуста официальный сайт` | Страница оффера или брендовая страница |
| Региональный | `займы в Омске`, `микрозайм в Москве` | Позже: региональная посадочная, только если есть реальная региональная логика |
| Бренд / МФО | `займер условия`, `webbankir займ` | Страница оффера с проверенными условиями |
| Условия займа | `займ 0 процентов`, `займ без процентов` | Подборка с осторожными формулировками и условиями акции |
| Отказ / просрочка / погашение | `почему отказали в займе`, `как продлить займ` | Практическая статья, чек-лист, FAQ |
| Калькуляторы | `калькулятор переплаты займа`, `рассчитать пск` | Сервисная страница `SERVICE` с инструментом |
| Сезонный / событийный | `займ перед зарплатой`, `займ на праздники` | Только если есть реальный спрос и польза, без агрессивных обещаний |

Найденные механики:

| Механика | Что дает | Что взять ZaimKarta |
| --- | --- | --- |
| Большие коммерческие подборки | Закрывают основной спрос и ведут к кликам | Подборки должны быть offer-first, с понятными условиями и фильтрами |
| Фильтры по сумме, сроку, возрасту, способу получения | Помогают пользователю выбрать | Делать постепенно, на базе реальных полей оффера |
| Осторожные страницы под рискованные запросы | Получают спрос, не обещая невозможного | `без отказа` трактовать как “повышенная вероятность”, а не гарантия |
| Проблемные статьи | Дают доверие и информационный трафик | Делать пошаговые материалы с правовыми источниками |
| Автор / редактор / дата обновления | Доверие и свежесть | Добавить в редакционный процесс и позже в админку статей |
| FAQ | Закрывает хвостовые вопросы | FAQ должен быть уникальным для страницы, а не шаблонным |
| Калькуляторы и сервисы | Повышают интерактивность и пользу | Развивать уже созданные `SeoTool` |
| Справочник / глоссарий | Покрывает базовые понятия | Сделать позже, после первых статей и подборок |

Слабости конкурентов, которыми можно пользоваться:

- слишком много одинаковых коммерческих блоков;
- рискованные обещания вроде “без отказа” без нормального объяснения;
- не всегда понятно, когда условия проверялись;
- статьи часто пересказывают очевидное и не дают конкретного выбора;
- мало связки “прочитал → проверил себя → перешел к подходящей подборке”.

## 5. Вывод для ZaimKarta

Самая сильная стратегия для ZaimKarta на ближайший этап:

1. Не штамповать статьи.
2. Сделать несколько сильных подборок под коммерческий спрос.
3. Добавить статьи, которые помогают принять решение и ведут к подборкам.
4. Использовать калькулятор и чек-лист как доверительный слой.
5. Делать факт-чек через ЦБ, закон и условия офферов.
6. Измерять не только трафик, но и переходы из статьи в подборку, клики по `/go/[slug]`, использование инструментов.
