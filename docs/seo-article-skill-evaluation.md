# ZaimKarta SEO Article Skill Evaluation

Дата проверки: 2026-07-20.

## 1. Задача

Нужно понять, можно ли использовать готовый skill для SEO-статей ZaimKarta или лучше сделать свой.

Важно: skill не должен автоматически публиковать статьи и не должен заменять редакторскую проверку. В финансовой теме это вспомогательный инструмент для исследования, плана, черновика и проверки рисков.

## 2. Что уже доступно в текущей среде

В текущей среде есть общие skills для документов, таблиц, презентаций, сайтов, визуализаций, OpenAI Docs, установки skills и создания skills.

Полезное для нас:

- `skill-creator` — можно создать собственный специализированный skill;
- `skill-installer` — можно установить готовый skill после отдельного согласования;
- `openai-docs` — полезен для вопросов по OpenAI/Codex, но не для SEO-статей;
- `zaimkarta-production-update` — полезен для выкладки на production, но не для написания статей.

Специализированного установленного skill именно под русскоязычные финансовые SEO-статьи ZaimKarta сейчас нет.

## 3. Что найдено среди готовых решений

Найдены общие решения:

- [Cody Article Writer](https://codyskills.ai/docs/cody-article-writer/getting-started/installation/) — общий article writer skill;
- [NanoSkill: SEO Article Writer](https://nanoskill.ai/skills/seo-writing) — общий SEO article writer;
- [AgriciDaniel/codex-seo](https://github.com/AgriciDaniel/codex-seo) — большой SEO audit skill suite для Codex;
- [openai/skills](https://github.com/openai/skills) — каталог skills и документация по подходу.

Вывод: это могут быть полезные ориентиры, но ни одно найденное решение нельзя безопасно взять “как есть” для ZaimKarta.

Причины:

- нет гарантированной адаптации под русский язык и Яндекс;
- нет встроенного понимания CPA-модели ZaimKarta;
- нет знания наших реальных страниц, офферов, `/go/[slug]`, `SeoPage`, `contentBlocks`;
- нет специальных запретов на рискованные микрофинансовые формулировки;
- нет обязательной проверки через ЦБ, 353-ФЗ и реальные условия офферов;
- могут быть лишние зависимости, внешние API, скрытая передача данных или платные сервисы;
- большой SEO-аудит-набор может быть избыточен для задачи “помочь писать безопасные статьи”.

## 4. Критерии отбора skill

Skill для ZaimKarta должен уметь:

- работать на русском языке;
- начинать с интента, а не с генерации текста;
- проверять текущую выдачу;
- использовать первоисточники;
- не выдумывать факты, цифры, ставки, рейтинги и условия;
- учитывать финансовую безопасность;
- запрещать гарантии одобрения;
- готовить `title`, `description`, `H1`, `H2/H3`;
- предлагать внутренние ссылки только на реальные страницы проекта;
- проверять дублирование темы внутри сайта;
- формировать список утверждений, которые нужно проверить вручную;
- выдавать результат в формате, совместимом с будущим SEO-редактором;
- не публиковать страницу автоматически;
- не устанавливать зависимости и не отправлять данные во внешние сервисы без разрешения.

## 5. Готовый skill или свой

| Вариант | Плюсы | Минусы | Решение |
| --- | --- | --- | --- |
| Готовый общий article writer | Быстро начать, есть структура | Не знает ZaimKarta, Яндекс, CPA, финансовые запреты | Не ставить без отдельной проверки |
| Большой SEO audit suite | Много SEO-функций | Избыточно, могут быть API/ключи/сложность | Не MVP |
| Собственный ZaimKarta skill | Учитывает продукт, источники, риски, админку | Нужно отдельно написать и протестировать | Лучший вариант |

Рекомендация: сделать собственный `zaimkarta-seo-article-writer` skill после того, как будет готов или хотя бы спроектирован редактор SEO-статей.

## 6. Что должен делать собственный skill

Skill должен работать так:

1. Принять тему статьи или кластер.
2. Проверить, нет ли уже похожей страницы в проекте.
3. Определить интент.
4. Проверить выдачу Яндекса и Google.
5. Найти первоисточники.
6. Составить план.
7. Подготовить `title`, `description`, `H1`, структуру `H2/H3`.
8. Написать черновик.
9. Добавить внутренние ссылки на реальные страницы ZaimKarta.
10. Сформировать FAQ.
11. Отдельно вынести утверждения, которые нужно проверить вручную.
12. Отдельно вынести рискованные места.
13. Указать дату факт-чека.
14. Не публиковать материал без владельца.

## 7. Формат результата skill

Каждый результат должен содержать:

- интент;
- целевой кластер;
- краткий анализ выдачи;
- источники;
- `title`;
- `description`;
- `H1`;
- план `H2/H3`;
- черновик статьи;
- FAQ;
- внутренние ссылки;
- ссылки на подборки/сервисы;
- список фактов для ручной проверки;
- список запрещенных или опасных формулировок;
- дату факт-чека;
- рекомендацию по следующему обновлению.

## 8. Пилотное тестирование

Первый тест должен быть только на одной статье.

Рекомендуемая пилотная тема:

`Что такое ПСК по займу и где смотреть полную стоимость кредита`

Почему эта тема хороша для теста:

- есть первоисточник в 353-ФЗ;
- тема важна для доверия;
- можно связать статью с калькулятором переплаты;
- нет необходимости обещать пользователю получение денег;
- можно проверить качество факт-чека.

Критерии успешного теста:

- текст понятен обычному человеку;
- нет выдуманных цифр;
- есть ссылки на источники;
- есть предупреждение о рисках;
- есть внутренняя ссылка на калькулятор и релевантную подборку;
- есть список фактов для ручной проверки;
- владелец может перенести материал в админку без технического разбора.

## 9. Что делать сейчас

Сейчас не устанавливать готовый skill.

Правильный порядок:

1. Сначала доработать редактор SEO-статей.
2. Затем написать ТЗ на собственный `zaimkarta-seo-article-writer`.
3. Потом протестировать skill на одной пилотной статье.
4. Только после этого решать, использовать его постоянно или доработать.
